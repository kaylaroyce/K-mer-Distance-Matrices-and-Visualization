# K-mer Distance Matrices and Data Visualization

## Owner
- Kayla Royce

The goal of this script is to analyze and compare nucleotide sequences using k-mer based methods. The script computes several distance matrices between FASTA sequences and visualizes sequence relationships using heatmaps and PCA.

<details>
  <summary><strong>Overview</strong></summary>
This bioinformatic pipeline analyzes user-specified nucleotide sequences by parsing FASTA files into k-mers of user-specified size and comparing their frequencies across all sequences. The scrip accepts multiple FASTA files as input and calculates pairwise sequence distances using several metrics. These comparisons help identify similarities and differences between sequences without relying on sequence alignment.

Outputs include:
- Distance matrix tables
- Pairwise comparison tables
- Heatmap visualizations
- PCA plots
- K-mer count tables for each sequence

_all code used for the pipeline can be found under "matrices-computation.py"_


</details>

<details>
  <summary><strong>Methods</strong></summary>

  <details>
    <summary>Input Data</summary>

   Requirements:
   - At least **two FASTA files**
   - Valid nucleotide characters (A,G,C,T)

   Command Line arguments are used to specify:
   - k-mer size
   - FASTA file paths
   - optional override flag for larger comparisons


  </details>
  <details>
    <summary>K-mer Parsing</summary>

Each sequence is parsed into overlapping k-mers of a specified size.

Tools used:
- custom Python parser

Process:
- FASTA sequences are read
- sequences are split into k-mers
- k-mer counts are recorded
- non-nucleotide characters are ignored

Output:
- dictionary containing k-mer counts for each sequence

  </details>

  <details>
    <summary>K-mer Normalization</summary>

K-mer counts are normalized by dividing each k-mer count by the total number of k-mers in the sequence.

Purpose:
- standardizes comparisons across sequences
- prevents bias due to sequence length differences

Output:
- normalized k-mer frequency dictionary

  </details>

  <details>
    <summary>Distance Matrix Calculations</summary>

Several distance metrics are calculated to compare sequences.

Tools used:
- custom Python functions

Distance metrics implemented:

- **Euclidean distance**
- **Manhattan distance**
- **Cosine dissimilarity**
- **Jaccard dissimilarity**

These metrics quantify sequence similarity using either frequency-based or presence/absence-based comparisons.

Output:
- `dm_output.tsv` containing pairwise sequence comparisons

  </details>

  <details>
    <summary>Pairwise Comparison Table</summary>

A cosine distance matrix is generated for all sequences.

Tools used:
- pandas

Output:
- `pairwise_compare.tsv`

This table shows the cosine dissimilarity between every pair of sequences.

  </details>

  <details>
    <summary>Heatmap Visualization</summary>

A heatmap visualization is generated from the pairwise cosine distance matrix.

Tools used:
- seaborn
- matplotlib

Output:
- `heatmap.png`

This heatmap visually represents similarity relationships between sequences.

  </details>

  <details>
    <summary>Principal Component Analysis (PCA)</summary>

PCA is performed on the k-mer count table to reduce dimensionality and visualize clustering patterns among sequences.

Tools used:
- pandas
- scikit-learn PCA
- matplotlib

Output:
- `pca.png`

The PCA plot shows how sequences cluster based on k-mer composition.

  </details>

</details>
